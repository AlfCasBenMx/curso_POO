🎓 Bloque 1 — Teoría: ¿Por qué HTML? (50 min)

🚀 ¿Por qué HTML en vez de PowerPoint y Word?

En la Sesión 1 usaron Excel y Power BI. Normalmente el siguiente paso sería Word (documento) y PowerPoint (presentación). Pero hay algo mucho más potente:

📽️ PowerPoint / Word

  • Estático — una vez exportado, no cambia
  • Requiere licencia de Office
  • Difícil de compartir sin adjuntos
  • Sin interactividad (no filtra, no calcula)
  • Diseño limitado al template
  • No funciona offline sin Office
  • Versionado difícil
vs

🌐 HTML (lo que haremos hoy)

  • Interactivo — filtros, colapsables, checkboxes
  • Gratis — se abre en cualquier navegador
  • Un link y todo el mundo lo ve (SharePoint, Teams, correo)
  • Responsivo — se adapta a celular, tablet, pantalla grande
  • Diseño libre — colores, grids, tarjetas, gráficos
  • Funciona offline — un .html en tu USB y listo
  • Versionado con Git — control total de cambios
💡 La clave: Ustedes no van a "programar" HTML. Van a pedirle a Copilot que lo genere. Copilot es EXCELENTE generando HTML porque es uno de los lenguajes con más ejemplos en su entrenamiento. El resultado suele ser de calidad profesional al primer intento.

🧬 Anatomía de un HTML (lo que necesitan entender — sin programar)

No van a escribir código. Pero entender 3 conceptos les permite dar mejores instrucciones a Copilot:

1. Estructura (HTML)

Las "piezas" de la página: títulos, párrafos, tablas, listas, tarjetas.

Piensen en HTML como los ladrillos de una casa.

<h1>Mi reporte</h1> <p>Hallazgos del análisis...</p> <table>...datos...</table>

2. Estilo (CSS)

Los colores, tamaños, espaciados, diseño visual.

CSS es la pintura y decoración de la casa.

body { font-family: Inter; } .card { background: #fff; border-radius: 12px; padding: 20px; }

3. Interactividad (JS)

Lo que hace que la página responda: filtros, botones, cálculos.

JavaScript es la electricidad de la casa.

// Al hacer click, filtra datos btn.onclick = () => { filtrar(datos, "Q1"); }
🎯 Lo que ustedes hacen: Describen en un prompt qué quieren ver → Copilot genera los 3 (HTML + CSS + JS) en un solo archivo → Copian → pegan en Notepad → guardan como .html → doble click → ¡listo!
⚡ Demo en vivo — El flujo completo en 60 segundos:
  1. Abrir copilot.microsoft.com
  2. Escribir: "Genera un HTML con una tarjeta que muestre: Total de gastos $4,500,000 MXN con fondo azul y texto blanco"
  3. Copilot genera el código → click en "Copiar"
  4. Abrir Notepad → Ctrl+V → Guardar como tarjeta.html (cambiar tipo a "Todos los archivos")
  5. Doble click en el archivo → se abre en el navegador ✨

⏱️ Todo el proceso toma < 60 segundos. Y el resultado es profesional.

🧠 Cómo pedir HTML a Copilot — Patrón de prompt

Un buen prompt para HTML sigue este patrón de 6 componentes:

TIPO DE PÁGINA + CONTENIDO + DATOS + ESTILO VISUAL + INTERACTIVIDAD + RESTRICCIONES
[TIPO] "Genera un HTML de una sola página tipo reporte ejecutivo" [CONTENIDO] "que muestre: Executive Summary, 4 KPIs en tarjetas, tabla de riesgos con semáforo, y recomendaciones" [DATOS] "Los KPIs son: Gasto Total $4.5M, Presupuesto $4.1M, Variación +9.7%, Departamentos en riesgo: 3" [ESTILO] "Estilo profesional corporativo, colores azul oscuro y amarillo (#FFCD00), fuente Inter, tarjetas con sombra" [INTERACTIVIDAD] "Agrega checkboxes en las recomendaciones, y que la tabla de riesgos sea ordenable por columna" [RESTRICCIONES] "Todo en un solo archivo HTML. Sin dependencias externas. Responsivo para móvil."

✅ Tips para mejores prompts HTML

  • Sé específico con los datos: da los números, no digas "datos de ejemplo"
  • Describe el layout: "4 tarjetas en fila", "dos columnas", "tabla abajo"
  • Pide colores concretos: "#FFCD00", "azul corporativo", "semáforo rojo/amarillo/verde"
  • Siempre pide "Todo en UN solo archivo HTML" — sin archivos CSS separados
  • Pide responsive: "Que funcione en móvil y desktop"
  • Itera: "Ahora cambia el fondo a oscuro" o "Agrega una sección de timeline"

❌ Errores comunes

  • "Hazme un HTML" (sin decir qué contenido)
  • No especificar los datos (Copilot inventa)
  • Pedir dependencias externas (jQuery, React) — complicación innecesaria
  • No pedir "en un solo archivo"
  • No describir el estilo visual
  • Guardar como .txt en vez de .html

💾 Flujo: de Copilot al navegador (paso a paso visual)

🤖 Copilot Chat
Escribir prompt → 📋 Copiar código
Click "Copy" → 📝 Notepad
Ctrl+V → Guardar → 🌐 Navegador
Doble click .html
⚠️ El paso que más falla: guardar como .html

En Notepad al guardar, por defecto guarda como .txt. Para guardar como HTML:

  1. Archivo → Guardar como
  2. En "Tipo" cambiar de "Documentos de texto" a "Todos los archivos (*.*)"
  3. En el nombre escribir: mi_reporte.html (con la extensión .html)
  4. En Codificación elegir UTF-8
  5. Click en Guardar
💡 Alternativa más fácil: Usar VS Code (si lo tienen instalado). Archivo nuevo → pegar → Ctrl+S → nombrar como .html y listo. VS Code incluso muestra el HTML con colores.

📄 Lab 1 — Reporte de resultados (reemplaza Word) — 25 min

Vamos a generar un HTML que sea el documento formal de su POC — con KPIs, tablas, riesgos y executive summary. Es lo que enviarían a un directivo.

Paso 1 — Adjuntar el Excel y pedir el reporte (5 min)

En Copilot Chat, adjuntar gastos_departamentales.xlsx (click en 📎) y escribir:

🤖 "Analiza este archivo Excel de gastos departamentales. Genera un HTML COMPLETO de una sola página tipo reporte ejecutivo con: 1. HEADER con logo de EY (usa la URL: https://companieslogo.com/img/orig/ernst-and-young-6c5de5ba.svg), título 'Reporte de Ejecución Presupuestal 2025' y fecha de hoy 2. EXECUTIVE SUMMARY — 5 bullets con los hallazgos más importantes (usa números reales del archivo) 3. 4 TARJETAS KPI en fila: - Gasto Total (suma de Gasto_Real_MXN) - Presupuesto Total (suma de Presupuesto_MXN) - Variación Promedio (promedio de Variacion_Pct, con color rojo si > 10%) - Registros con variación > 20% (conteo) 4. TABLA de gastos por departamento: Departamento, Total Presupuesto, Total Gasto Real, Variación %, con semáforo (🟢 < 10%, 🟡 10-20%, 🔴 > 20%) 5. TABLA DE RIESGOS con 5 riesgos: Riesgo, Probabilidad, Impacto, Mitigación 6. SECCIÓN DE RECOMENDACIONES con 4 recomendaciones específicas basadas en los datos, con checkboxes interactivos Estilo: profesional corporativo. Colores: fondo blanco, acentos en azul (#0F62FE) y amarillo (#FFCD00). Fuente: Inter o system-ui. Tarjetas con sombra y bordes redondeados. Responsivo. TODO en un solo archivo HTML, sin dependencias externas. Incluye CSS y JavaScript inline."
Paso 2 — Copiar, guardar y abrir (3 min)
  1. Copilot genera el código HTML completo
  2. Click en "Copiar" (o seleccionar todo el código)
  3. Abrir Notepad → Ctrl+V
  4. Guardar como: reporte_presupuestal.html (Tipo: Todos los archivos, Codificación: UTF-8)
  5. Doble click en el archivo → se abre en el navegador
Paso 3 — Iterar y mejorar (7 min)

Vuelve a Copilot Chat y pide mejoras. No empieces de cero — itera:

🤖 "Modifica el HTML anterior: - Agrega una sección de 'Cronograma del POC' con 3 fases en un timeline visual horizontal - En la tabla de departamentos, agrega una columna de 'Tendencia' con flechas ↑ ↗ → ↘ ↓ - Haz que las tarjetas KPI cambien de color si la variación supera el 10% - Agrega un footer con 'POC Thinking — EY · Confidencial'"
🔄 Plan B (si Copilot da solo el fragmento modificado): Pide: "Dame el HTML COMPLETO con todas las modificaciones, no solo el fragmento."
Paso 4 — Personalizar con datos de tu equipo (10 min)

Adjunta el dataset personalizado de tu equipo (o describe tu problema) y pide:

🤖 "Adapta este reporte HTML para nuestro POC: - Título: 'Reporte del POC: [NOMBRE DE TU POC]' - El problema es: [DESCRIPCIÓN DE TU FICHA DE PROBLEMA] - Cambia los KPIs a los que son relevantes para nuestro caso: [LISTA TUS KPIS] - En los riesgos, incluye: [TUS RIESGOS ESPECÍFICOS] - Mantén el mismo estilo visual Dame el HTML completo."

📊 Lab 3 — Dashboard interactivo (nivel avanzado) — 20 min

El más impresionante: un mini-dashboard con datos reales, filtros y gráficos que funciona 100% offline en el navegador.

Paso 1 — Generar el dashboard con datos del Excel (8 min)

Adjunta gastos_departamentales.xlsx a Copilot Chat:

🤖 "Analiza este archivo Excel. Extrae los datos y genera un HTML tipo dashboard ejecutivo con los datos reales embebidos en JavaScript. El dashboard debe tener: 1. HEADER con título 'Dashboard de Control Presupuestal 2025' y filtro dropdown para seleccionar Departamento (opción 'Todos' + cada departamento del archivo) 2. FILA DE KPIs (4 tarjetas): Gasto Total, Presupuesto Total, Variación Promedio, Total Registros. Los valores deben ACTUALIZARSE cuando se filtra por departamento. 3. TABLA de datos con columnas: Departamento, Categoría, Mes, Presupuesto, Gasto Real, Variación %. La tabla debe: - Filtrarse cuando se selecciona un departamento - Tener filas coloreadas: verde si variación < 10%, amarillo 10-20%, rojo > 20% - Ser ordenable por click en el header de columna 4. GRÁFICO de barras horizontales mostrando Gasto Real por Departamento (usar divs con CSS como barras, no necesita librería externa). Las barras deben tener el valor numérico a la derecha. Paso 2 — Guardar y probar interactividad (3 min)

Guardar como dashboard_presupuestal.html. Abrir y probar:

  • ¿Los KPIs cambian al filtrar?
  • ¿La tabla se colorea por semáforo?

Paso 3 — Agregar más interactividad (9 min)
🤖 "Mejora el dashboard: 1. Agrega un segundo filtro por Mes (dropdown) 2. Agrega una barra de búsqueda que filtre la tabla en tiempo real 3. En los KPIs, muestra una flecha ↑ o ↓ y el porcentaje de cambio respecto al mes anterior 4. 🎯 Para la Sesión 3: ¿Qué mostramos?

Cada equipo tendrá mínimo 4 artefactos para su demo final:

  1. 📊 Excel enriquecido — datos analizados con Copilot
  2. 📈 Dashboard Power BI — visualización ejecutiva
  3. 📄 Reporte HTML — documento formal interactivo
  4. 📽️ Presentación HTML — pitch visual para el comité

Bonus: 📊 Dashboard HTML interactivo como quinto artefacto.

📦 Entregable de la Sesión 2

  1. Reporte HTML del POC — personalizado con datos de su equipo. KPIs, riesgos, métricas, recomendaciones.
  2. Presentación HTML del POC — mínimo 6 slides con problema, solución, antes/después, timeline, Call to Action.
  3. (Bonus) Dashboard HTML — con datos reales/simulados, filtros y semáforos.

💡 Guardar todos los .html en la carpeta mi_poc_html. En la Sesión 3 los abrirán en el navegador frente al grupo para el pitch final.

📋 Preparación para la Sesión 3

Sesión 3: Demo Final y Pitch

Cada equipo presentará su POC completo ante el grupo (simulando un comité directivo).

Lo que deben preparar:

  1. Incorporar feedback de la revisión cruzada en sus HTMLs.
  2. Practicar la demo en vivo (5-7 min): abrir la presentación HTML → mostrar slides → cambiar a dashboard → mostrar interactividad → mostrar reporte.
  3. Ensayar la narrativa: ¿cómo convencerían a un directivo en 5 minutos?
  4. Preparar el "wow moment": un filtro que revela algo sorprendente, un antes/después impactante, un número que cambia todo.
  5. Tener backup: Si algo falla, tener screenshots de los HTMLs listos.

Formato de la Sesión 3:

BloqueActividadDuración
Bloque 1Últimos ajustes + ensayo del pitch45 min
Bloque 2Demo + Pitch por equipo (5-7 min) + Q&A del "comité"90 min
Bloque 3Feedback, lecciones aprendidas, próximos pasos reales45 min

🚨 Troubleshooting — Problemas comunes

ProblemaSolución
El archivo se abre como texto, no como páginaVerificar que la extensión sea .html y no .html.txt. En Notepad, guardar con Tipo = "Todos los archivos".
Se ven caracteres raros (ñ, acentos)En Notepad al guardar, cambiar Codificación a UTF-8. Verificar que el HTML tenga <meta charset="utf-8">.
Copilot no da el HTML completoPedir: "Dame el código HTML COMPLETO desde <!doctype html> hasta </html>, todo en un bloque de código."
Copilot da solo el fragmento modificado"Dame el HTML COMPLETO con todas las modificaciones incorporadas, no solo el fragmento."
Los filtros del dashboard no funcionanPedirle a Copilot: "Los filtros no funcionan. Revisa el JavaScript y corrige. Dame el HTML completo corregido."
Los datos del Excel no aparecen en el HTMLAsegurarse de adjuntar el archivo Excel en Copilot Chat. Si no lo toma, copiar 20-30 filas y pegarlas en el prompt.
El HTML se ve mal en el celularAgregar al prompt: "Hazlo responsivo con media queries para pantallas pequeñas."
Quiero cambiar colores o layoutNo hace falta empezar de cero: "Cambia la paleta de colores a [X]. El layout de las tarjetas debe ser 2x2 en vez de 4 en fila."
El HTML es muy largo para CopilotDividir en partes: "Primero genera el header y los KPIs. En el siguiente mensaje te pido el resto."
Quiero usar Chart.js u otra libreríaPedir: "Incluye Chart.js vía CDN y genera un gráfico de barras con los datos." Funciona, pero requiere internet para cargar la librería.

⚡ Cheatsheet — Prompts clave para generar HTML

📄 Reportes

"Genera un HTML de reporte ejecutivo con: header corporativo, executive summary, [N] tarjetas KPI, tabla de [datos] con semáforos, sección de riesgos, recomendaciones con checkboxes. Estilo: [colores]. Todo en un archivo."

📽️ Presentaciones

"Genera un HTML de presentación con [N] slides (secciones 100vh). Slide 1: portada. Slide 2: problema. Slide 3: solución. [...]. Navegación con puntos fijos. Scroll suave. Estilo: [colores]. Todo en un archivo."

📊 Dashboards

"Genera un HTML tipo dashboard con filtro por [campo], KPIs que se actualicen al filtrar, tabla con semáforos, gráfico de barras CSS. Datos embebidos en JavaScript: [pegar datos o adjuntar Excel]. Todo en un archivo."

🔄 Iteraciones comunes

"Cambia la paleta de colores a dark mode."
"Agrega una sección de timeline horizontal."
"Haz las tarjetas más grandes con íconos."
"Agrega un botón para exportar la tabla a CSV."
"Pon una barra de búsqueda que filtre la tabla."
"Dame el HTML completo con todos los cambios."
