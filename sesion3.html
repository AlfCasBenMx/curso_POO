Sesión 3 — Laboratorio Python I

Instalación del entorno y primer script — preparar la máquina para los laboratorios.

Sesión C · Laboratorio 16-dic-25 · 15:00–16:00 (México)

Objetivo del laboratorio

Dejar listo el entorno de desarrollo en Windows para ejecutar los ejemplos del curso: instalar Python, configurar un entorno virtual, instalar dependencias y ejecutar scripts sencillos.

  • Verificar o instalar Python 3.
  • Instalar un editor recomendado (VS Code) y extensiones mínimas.
  • Crear y activar un entorno virtual.
  • Instalar dependencias con pip desde requirements.txt si aplica.
  • Ejecutar un primer script de prueba.

Agenda rápida (60 min)

  1. Comprobación previa y preguntas (10 min)
  2. Instalación y configuración (25 min)
  3. Primer script y pruebas (15 min)
  4. Preguntas y cierre (10 min)

Pasos prácticos

1) Verificar Python

Abre una ventana de cmd.exe y ejecuta:

python --version

2) Crear y activar un entorno virtual (Windows, cmd.exe)

python -m venv .venv
.venv\Scripts\activate

3) Actualizar pip e instalar dependencias

python -m pip install --upgrade pip
python -m pip install -r requirements.txt

4) Ejecutar un script de prueba

Crea un archivo hola.py con este contenido y ejecútalo:

print('Hola desde Python - Sesión 3')
python hola.py

Checklist rápido de preparación

  • Python 3 instalado y accesible desde la línea de comandos.
  • Editor instalado (VS Code recomendado).
  • Entorno virtual creado y activado.
  • Dependencias instaladas sin errores (si existe requirements.txt).
  • Primer script ejecutado correctamente.

Resolución de problemas comunes

  • Si python no se reconoce: instala Python y marca "Add to PATH" en el instalador.
  • Si la activación del entorno falla en PowerShell, ejecuta: Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned en PowerShell (admin) o usa cmd.exe.
  • Errores al instalar paquetes: revisa la versión de pip y vuelve a intentarlo; revisa conexión a internet y permisos.

Revisiones y checklist de validación (para el laboratorio)

Antes de aceptar resultados (scripts o fórmulas), pide a los participantes que sigan esta lista rápida de validación.

  1. Prueba con un subconjunto pequeño (3–5 filas) y calcula manualmente para verificar el total.
  2. Comprueba tipos de dato: fechas como fechas, importes como números (sin comas ni símbolos).
  3. Verifica que los valores nulos o vacíos no rompen el cálculo (usa IFERROR o condiciones explicitas).
  4. Haz una comparación entre la versión en Python y la versión en Excel: sumar totales y comparar diferencias porcentuales (deben ser 0 o muy pequeñas por redondeo).
  5. Revisa los casos límite: montos 0, fechas limítrofes (inicio/fin de periodos), categorías desconocidas.
  6. Documenta brevemente (1–2 líneas) la lógica usada y los supuestos (por ejemplo: "Se ignoran importes negativos").

Prompts de ejemplo para pedir revisiones al asistente de IA:

  • "Haz una revisión de la fórmula y enumera posibles errores de entrada que la harían fallar."
  • "Explica en lenguaje de negocio por qué el resultado difiere en 0.2% entre Python y Excel y sugiere acciones para corregirlo."
  • "Genera pasos concretos para convertir este cálculo en una tabla dinámica reproducible en Excel."

Lista de ejercicios (los trabajaremos uno a uno)

  1. Hola — lectura rápida de un script simple (sin archivo).
  2. Sumar importes — usa transacciones.csv (columna importe): =SUM(...)
  3. Contar operaciones grandes — usa ventas.csv (columna importe): =COUNTIFS(...)
  4. Aplicar IVA — usa ventas.csv o transacciones.csv: crea importe_con_iva =C2*1.16
  5. Sumar por categoría — usa ventas.csv (columnas categoria, importe): =SUMIFS(...)
  6. Agrupar y sumar — ventas.csv: tabla dinámica o SUMIFS por categoría.
  7. Lookup entre hojas — ventas.csv + clientes.csv: añade cliente_nombre con XLOOKUP/VLOOKUP.
  8. Regla escalonada — ventas.csv: columna impuesto con IFS o IF anidados.
  9. Verificación Python vs Excel — ventas.csv + tarifas.csv: calcula impuesto por categoría y compara con el resultado en Python.
  10. Asistente IA para fórmula — usa ventas.csv: pide al asistente la fórmula SUMIFS por categoría y rango de fechas y pruébala.
